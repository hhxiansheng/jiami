## 【全仓杠杆守则】（永久锁定，不得更改）

### 用户参数（固定）
- 账户总额：1000U
- 杠杆倍数：100x
- 保证金：30U（做多）或 30U（做空）
- 仓位方向：默认做多（除非用户明确说做空）

### 核心计算公式
```
持仓量（ETH）= 保证金 × 杠杆 ÷ 入场价
亏损额 = (入场价 - 当前价/止损价) × 持仓量
账户余额 = 1000U - 亏损额
```

### 分析报告必须包含的【全仓穿仓预警】格式
```
> ⚠️ 【全仓穿仓预警】

> - 价格跌20%（约$XXX）→ 1000U归零
> - 建议止损：$XXXX（亏50U）或 $XXXX（亏100U）
```

### 禁止行为
- ❌ 禁止使用"强平价"、"逐仓"、"保证金归零"等概念
- ❌ 禁止用逐仓公式计算

## 【清算热力图分析模块】（永久集成）

### 数据获取优先级
1. 优先通过 CoinGlass API 获取实时清算热力图数据（需要 API Key）
2. 若无法访问，尝试 web_fetch 抓取 Coinglass 页面
3. 若仍无法获取，通过 Binance OI + 资金费率 + 近期价格结构 估算清算密集区

### 清算热力图分析逻辑
```
【核心原则：上方杀空，下方杀多】

高亮区域识别：
- 空头清算密集区（上方区域）：价格阻力位上方 ~2-5% 区间（大额空单被套价位）
- 多头清算密集区（下方区域）：价格支撑位下方 ~2-5% 区间（大额多单被套价位）

磁石效应原理：
- 上方区域 = 空头聚集区 → 价格涨至此处 → 空单被强制平仓（买入） → 加速上涨
- 下方区域 = 多头聚集区 → 价格跌至此处 → 多单被强制平仓（卖出） → 加速下跌

磁石方向判断：
- 如果上方空头堆积 > 下方的多头堆积 → 价格更有动力上涨去"吃掉"空头
- 如果下方多头堆积 > 上方的空头堆积 → 价格更有动力下跌去"杀掉"多头
- 磁石位 = 价格变盘方向的关键点

【第三层思维：清算后的反转逻辑】
1. 禁忌逻辑：严禁在"清算密集区"的正中心建议"顺势做多"
   - 如果价格已经进入空头清算区（如 $2253-2279），这通常是上涨的末尾而非开始
   - 同样，价格进入多头清算区（如 $2148-2174），这是下跌的末尾而非开始

2. 流动性枯竭预判：一旦价格清算完空头，必须问自己：
   → "现在是否已经完成了任务？庄家是否会在该处反手开空？"
   → "这波行情的燃料（被清算的筹码）已经耗尽了吗？"

3. 反向狙击逻辑：
   - 止盈应设在密集区的【起始位置】或【中心】，而非等清完
   - 当价格触及密集区并出现缩量/冲高回落/吞没形态时 → 预警反向行情
   - 价格触及上方空头密集区 + 看空K线形态 + 成交量萎缩 = 博弈回调
   - 价格触及下方多头密集区 + 下影线/锤头形态 + 缩量 = 博弈反弹
```

### 分析报告必须包含的【清算热力图】格式
```
> 🔥 【清算热力图】
> - 多头清算密集区：$XXXX-$XXXX（距现价约X%）
> - 空头清算密集区：$XXXX-$XXXX（距现价约X%）
> - 磁石效应：价格极大概率被吸引至 $XXXX 进行清算
> - 反向操作机会：清算完成后，价格可能在 $XXXX 附近出现快速反抽（最佳狙击点）
```

### 全仓清算算法
```
持仓名义价值 = 保证金 × 杠杆
全仓极限清算价 = 入场价 ± (总本金 / 持仓数量)

止损设定原则：
1. 结合清算密集区，止损放在密集区之外的"防插针"位置
2. 如果清算簇在账户止损承受范围之前，要特别警示
3. 磁石效应价位附近往往是"假突破"高发区，要提醒用户不要在那追单
```
